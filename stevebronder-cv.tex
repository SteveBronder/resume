\section{\faCubes{} SELECTED PROJECTS}

\project{https://github.com/SteveBronder/stancon2018}{\texttt{stanMathCL}: GPU Routines for the Stan Programming Language}{
To be presented at Stancon 2018 \par
 This work is the first known open source implementation of the Cholesky decomposition using a GPU in a Hamiltonian Monte Carlo setting. Results show that large matrices can see speedups of 8x while retaining the same precision as CPU methods.
}

\project{https://rawgit.com/Stevo15025/mlrForcastingTutorial/gh-pages/forcasting_in_mlr.html}{\texttt{mlrfc}: An R package for forecasting with machine learning models}{
Presented at R in Finance 2017: \href{https://channel9.msdn.com/Events/RFinance/RFinance-2017/Lightning-Talks-I-Day-2}{(click here for video, I am the first speaker})\par
 R package combining methodologies of machine learning with classical forecasting techniques. Combining these models with machine learning methods for tuning hyperparameters makes for more robust models. This package can also use machine learning methods for forecasting.

}

\project{https://github.com/stevebronder/tsp_gpu}{\texttt{Columbus}: GPU Routines for solving the traveling salesman problem}{
Presented at Columbia Electrical Engineering Student Conference (2016) \par
Columbus uses CUDA to implement a multi-core simulated annealing routine for solving large ($N>100K$) Euclidian TSPs. This software can come within 3\% of the optimal path with a 30x speed increase.
}
